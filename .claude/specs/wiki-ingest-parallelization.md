# 設計書：wiki-ingest 並列化

## 概要

`wiki-ingest` の処理を逐次実行から4フェーズ構成に変更し、tool call の並列発行によって処理時間を削減する。

---

## 変更対象ファイル

| ファイル | 変更内容 |
|----------|----------|
| `CLAUDE.md` | インジェストセクションを4フェーズ構成に書き換え |
| `.claude/commands/wiki-ingest.md` | フェーズ構成への参照を追加 |

---

## 現状の問題

1 ソースあたり以下が逐次発火する：

- rawファイル読み込み（1 tool call）
- `ls concepts/` `ls entities/` 個別チェック（複数 tool calls）
- concept/entity ページを1つずつ読んで更新（10〜15 tool calls × 逐次）
- index.md 更新、log.md 更新

合計 **20〜30 tool calls** が逐次実行されることで処理時間が長くなっている。

---

## 新インジェスト手順（5フェーズ）

### フェーズ0 — 計測開始

```bash
date +%s
```

開始時刻（Unix秒）を記録する。

### フェーズ1 — 並列収集

以下を**同時に**実行する（tool call を並列発行）：

- rawソースファイルを読み込む
- `wiki/concepts/` のファイル一覧を取得
- `wiki/entities/` のファイル一覧を取得
- `wiki/index.md` を読み込む
- 既存ページへの追記が必要な場合は、該当ファイルの読み込みもここで並列実行

### フェーズ2 — 一括設計（tool call なし）

LLM 内部で以下を一括決定する：

- rawファイルに登場するコンセプト・エンティティを全列挙
- summaryページの内容を確定
- 各 concept/entity ページの内容を確定（新規作成 or 既存への追記）
- 既存ページとの矛盾箇所を特定

**デフォルトは確認なしで処理を進める。** ユーザーが「確認しながら進めて」と指示した場合のみ、このフェーズ終了後に列挙内容を提示して確認する。

### フェーズ3 — 並列書き込み

以下を**同時に**実行する（tool call を並列発行）：

- `wiki/summaries/[YYYY-MM-DD-slug].md` を作成（同名が既存の場合は末尾に連番。例: `2026-04-12-llm-wiki-2.md`）
- 各 concept ページを新規作成または追記
- 各 entity ページを新規作成または追記

**必ず raw/ ソースファイルを直接参照して書くこと。summary から派生させない。**  
既存ページの内容と矛盾する場合は矛盾を明記する。

### フェーズ4 — 逐次更新

書き込み内容が確定してから順に実行する：

1. `wiki/index.md` を更新
2. `date +%s` で終了時刻を取得し経過時間を計算する
3. `wiki/log.md` に追記：`## [YYYY-MM-DD] ingest | [ソースタイトル] | ⏱ Xs`

---

## 効果の見込み

| 指標 | 改善前 | 改善後 |
|------|--------|--------|
| フェーズ1の tool calls | 逐次 4〜6 回 | 並列 1 ラウンド |
| フェーズ3の tool calls | 逐次 10〜15 回 | 並列 1 ラウンド |
| 合計ラウンド数（目安） | 20〜30 回 | 5〜8 回 |
